A — Atomicité : tout ou rien. Si une étape échoue, toutes les écritures sont annulées.

C — Cohérence : la base reste dans un état valide avant et après chaque transaction.

I — Isolation : les transactions concurrentes ne se voient pas mutuellement avant le commit.

D — Durabilité : après un commit, les données survivent aux pannes système.
