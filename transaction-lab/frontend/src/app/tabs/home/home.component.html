Kata pédagogique

Transaction Lab

Explorez les patterns de transactions distribués en conditions réelles — chaque module est un kata interactif avec Spring Boot et Angular.

Java 21 Spring Boot 3.3 Angular 18 Kafka

Pourquoi ce lab ?

Dans un système distribué, garantir la cohérence des données est l'un des défis les plus sous-estimés du développement backend. On le lit dans les livres. On le comprend en le cassant.

Que se passe-t-il si le serveur tombe entre le débit et le crédit ?

Comment Redis choisit entre cohérence et disponibilité lors d'une partition ?

Comment annuler une commande qui a déjà déclenché 3 micro-services ?

Ce lab te fait vivre les pannes, pas juste les lire.

Objectif pédagogique

Chaque module suit l'approche kata : manipulez le système jusqu'à provoquer la panne, observez la réaction, comprenez pourquoi. Les guides kata dans le dépôt vous accompagnent étape par étape pour ancrer les concepts ACID, CAP et SAGA dans votre mémoire musculaire de développeur.
