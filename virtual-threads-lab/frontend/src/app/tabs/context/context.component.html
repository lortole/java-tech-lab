Context Propagation

Quand un Virtual Thread enfant est créé, le MDC (ThreadLocal) du thread parent n'est pas propagé. Votre trace ID disparaît silencieusement dans les logs.

Le problème

Dans une app web, le contexte transporte souvent un traceId, un utilisateur courant, une locale ou des informations de sécurité. Si une tâche part dans un autre virtual thread sans propagation, les logs deviennent difficiles à relier.

Ce qui se passe au fork

Thread parent
traceId=abc-123
fork
Virtual thread
traceId=null

Un ThreadLocal appartient au thread qui le porte. Créer un VT ne copie pas automatiquement son contenu.

La bonne question

Le sujet n'est pas seulement "est-ce que le code compile ?". C'est : est-ce que le contexte métier et observabilité traverse les frontières de concurrence ?

ThreadLocal seul simple, mais perdu dès qu'on change de thread
Copie manuelle MDC fonctionne, mais demande discipline et cleanup
ScopedValue modèle explicite, immuable, adapté à la concurrence structurée
Micrometer pratique dans Spring Boot pour propager sans polluer le code métier
